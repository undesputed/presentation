05
Why MCP

Why we
chose MCP.

MCPを選んだ理由
Why Model Context Protocol?
Not a custom integration for each tool — a universal standard. Here's why that matters. 各ツールへの独自統合ではなく、普遍的な標準プロトコル
🔌

One Standard, Any Tool

一つの標準、あらゆるツール

MCP is an open protocol by Anthropic. Instead of writing a custom integration per tool, the AI speaks one language that any MCP-compatible tool understands. Adding a new tool means one new server — not a rewrite.

🔒

Secure by Design

設計段階からセキュア

Each MCP server is a controlled gateway. The AI never touches credentials or raw APIs directly — it receives only what the server explicitly allows. Access is permission-scoped and auditable.

👥

Zero Team Disruption

チームへの影響ゼロ

Teams keep using Backlog, Slack, and Calendar exactly as before. MCP reads and writes in the background — the only experience that changes is the PM's.

⚡

Real-Time, Not Cached

リアルタイムデータ

Every MCP call fetches live data from the source system. The AI always answers with the current state of your sprint — not a stale snapshot from hours ago.

How MCP Works

Inside
the flow.

MCPの仕組み — このプロジェクトで
How MCP Works in This Project
Press → to walk through each step → キーで各ステップを表示
1

PM asks: "What's blocking the current sprint?"

A natural language question typed into the chat interface — no tool navigation or knowledge of the underlying systems needed.

2

AI interprets the intent

The AI understands the goal — retrieve blocked tasks from the active sprint. It selects the right MCP server to call.

Backlog MCP selected
3

MCP server calls the Backlog API

The Backlog MCP server translates the AI's request into a real API call and returns structured task data — securely, with no credentials exposed to the model.

GET /api/v2/issues?projectId=X&statusId=blocked
4

AI synthesizes an answer

Instead of raw JSON, the AI generates a readable summary — task names, owners, how long each has been blocked, and suggested next steps.

5

PM gets a clear, actionable answer — instantly

"2 tasks are blocked: [Task A] waiting on API spec · 3 days, Kenji  ·  [Task B] pending design review · 1 day, Aoi."

No tabs opened · No manual checking
